# Anthropic Engineers Don't Code Anymore

## Summary

Alex recommends AI implementation as a business and says people can often skip some
traditional coding basics by using AI tools directly. He describes, as hearsay, an
understanding that Anthropic engineers manage swarms of agents and spend more time in
project management and code review than hands-on coding. His broader advice is to start
with the customer and problem, then learn tools in the context of solving that problem.

## Key claims (Alex's advice only)

- Alex recommends AI implementation as a business.
- He says AI is the first product that can teach users how to use it.
- Alex says, as hearsay, that Anthropic engineers may use swarms of agents and focus on
  project management and code review.
- The learner should start with a customer and problem rather than a vehicle or tool.
- Tool learning becomes more useful when tied to a concrete problem.
- Alex uses the wrench analogy: learning a tool makes more sense after there is an
  engine to open.

## Notable verbatim quotes

> "AI is the first product that has ever existed that will teach you how to use it."

> "Start with the problem."

> "What customer do you want to serve? What problem do you want to solve?"
